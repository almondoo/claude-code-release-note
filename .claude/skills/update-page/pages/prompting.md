# prompting — プロンプト

## メタデータ

| 項目 | 値 |
|------|-----|
| ページ種別 | ガイド型 |
| アイテム数目安 | 約30件（7セクション） |
| WebFetch 欠落リスク | 低（ソースURL1つ） |

### 高リスクフィールド

- `examples[].before` / `examples[].after`: プロンプト例。具体的な文面が要約で変わりやすい
- `code[].value`: コード例。要約で省略されやすい

## データファイル

`app/data/prompting/prompting.json`

## 情報ソース

| ステップ | 内容 | ソース URL |
|---------|------|-----------|
| 1. 公式ドキュメント取得 | プロンプトのベストプラクティス | `https://code.claude.com/docs/en/best-practices` |
| 2. 差分確認 | 既存データとの比較 | ローカルの `prompting.json` |

## データスキーマ

### 全体構造

```json
{
  "sections": [
    {
      "id": "section-id",
      "name": "セクション名",
      "description": "セクションの説明",
      "items": [...]
    }
  ]
}
```

### アイテム

```json
{
  "id": "item-id",
  "title": "日本語タイトル",
  "summary": "1〜2行の要約",
  "content": "詳細解説（\\n\\n で段落分け）",
  "tags": ["タグ"],
  "examples": [
    {
      "strategy": "戦略名",
      "detail": "補足説明（オプショナル）",
      "before": "悪い例のプロンプト",
      "after": "良い例のプロンプト"
    }
  ],
  "tips": ["ヒント1", "ヒント2"],
  "code": [
    { "lang": "python", "label": "ラベル", "value": "コード文字列" }
  ]
}
```

#### フィールド詳細

| フィールド | 必須 | 型 | 説明 |
|-----------|------|-----|------|
| `id` | 必須 | string | 一意識別子 |
| `title` | 必須 | string | 日本語タイトル |
| `summary` | 必須 | string | 1〜2行の要約 |
| `content` | 必須 | string | 詳細解説。`\n\n` で段落分け |
| `tags` | 必須 | string[] | タグ配列（空配列可） |
| `examples` | オプショナル | array | before/after 形式のプロンプト例 |
| `tips` | オプショナル | string[] | 実践的なヒントのリスト |
| `code` | オプショナル | object[] | **配列オブジェクト形式**。`lang`, `label`, `value` を持つ |

**注意:** `code` フィールドは配列オブジェクト形式 `[{ lang, label, value }]` であり、skill-best-practices の string 型（マークダウンコードブロック）とは異なる。

## ページ固有ルール

### プロンプト例の記述

- `before` / `after` は実際のプロンプト例を記述する（抽象的な説明ではなく具体的なプロンプト文）
- プロンプト内のコード部分（Python, bash 等）やコマンド名はそのまま英語で記述
- プロンプトの説明・指示部分は日本語に翻訳
- `strategy` は比較の観点を簡潔に表す日本語

### 固有翻訳ルール

- `content` は2〜3段落で、概念の説明・具体的な適用方法・注意点を記述
- `tips` は実践的で短いアドバイス（1文程度）

## 使用条件・注意事項

### 使用条件

- 公式プロンプトガイダンスに変更・追加があったとき
- 既存のプロンプトテクニックの説明や例が更新されたとき

### 使わないケース

- UI/デザインの修正
- 既に正確に存在するアイテムの再追加

## Common Mistakes

| ミス | 対処 |
|------|------|
| `before` / `after` のプロンプト例が抽象的・不自然 | 実際にユーザーが入力しそうな具体的なプロンプト文にする |
| `code` を string 型で記述 | prompting では配列オブジェクト形式 `[{ lang, label, value }]` を使用 |
| `content` の段落を `\n` で区切る | `\n\n` を使用（ParagraphList の仕様） |
| プロンプト例のコード部分まで日本語に翻訳 | コード構文・コマンド名はそのまま英語で記述 |
